Отчет о проделанной работе по изучению Git
Дата: 29.10.2025

Чему я научился:
Понимание основ: Я разобрался, что Git — это не просто хранилище файлов, а система для отслеживания изменений. Я четко понимаю разницу между моим локальным репозиторием на компьютере и удаленным на GitHub.

Работа в терминале: Я освоил работу с Git через командную строку PowerShell, что дает мне полное понимание и контроль над процессом.

Ключевые рабочие циклы:
Создание проекта с нуля: Я научился инициализировать локальный репозиторий (git init), связывать его с удаленным (git remote add) и делать первую выгрузку (git push -u origin main).
Работа с существующим проектом: Я освоил клонирование репозитория (git clone) и последующую работу с ним.

Ежедневный ритуал: Цикл git add -> git commit -> git push стал для меня понятным и привычным.

Решение проблем и отладка: Это самый важный опыт. Я научился не бояться ошибок, а читать их и исправлять:
Я знаю, что ошибка [rejected] при push означает расхождение историй и решается командой git pull.

Настройки и кастомизация:
Я понимаю разницу между глобальной и локальной конфигурацией (git config) и на практике научился делать коммиты от разных "авторов", чтобы увидеть эту разницу в действии.

Столкнулся с проблемой, гит не добавляет файлы в коммит потому что файл не претерпел изменения, соответственно и не зачем его еще раз выгружать.

31.10.25 столкнулся с тем что нужно было править коммит, поправил его через команду git reset --soft HEAD~1.
